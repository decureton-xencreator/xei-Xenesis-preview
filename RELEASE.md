## Category

Xen is the operating system for designing Living Companies.

## Production replacement

The prior prototype used one large scene script with later patch runtimes layered after it. This release replaces that execution path with a single production runtime under `src/`, a session-safe application shell, embedded Living Manual Runtime, deterministic manual router, contextual revision and approval experience, propagation visualization, personalized Executive Summary, company and department selectors, and Blueprint Session confirmation.

## Canonical proof represented

The premiere preserves the approved Checkmate BDC origin: there was no BDC department; Marisol was a virtual assistant used as the de facto BDC; Vianka Gomez became the first actual BDC Manager; Vianka and five Panamanian agents had no kitchen cabinet or countertop sales experience; and the BDC publication family converted operational knowledge into repeatable, inheritable infrastructure.

The embedded family includes XBM-101 Agent, XBM-102 Manager, XBM-103 Objection, and XBM-104 Script operating content currently present in the repository proof experience. It no longer opens a separate manual node.

## Runtime controls

- Progress persists to `localStorage` on every answer, input event, visibility change, and page exit.
- No iPhone Safari Web Speech recognition dependency exists.
- Apple keyboard dictation is explicitly guided.
- Browser synthesis is optional and never blocks scene progression.
- Manual reading scenes contain no narration.
- Runtime errors fall into a recoverable session boundary.
- Mobile safe-area controls and reduced-motion accessibility are included.

## Validation

`npm test` executes static certification gates covering the application shell, 18-scene autonomous flow, persistence, recovery, Safari-safe input, embedded manual viewing, canonical family presence, search, routing, propagation, summary generation, selectors, mobile CSS, and accessibility baseline.

GitHub Actions runs the suite on every push and pull request.

## Certification truth

This repository release is a production-quality release candidate. Diamond/Gold Master certification requires the GitHub Actions validation run plus device acceptance evidence on real iPhone Safari and desktop browsers. A live neural voice provider is intentionally external to canonical source and remains a future XVR production integration dependency.
